# 🚀 100 Days of Applied Data Science
<big> current status : Day 10 completed <big/>
Welcome to my **100 Days of Applied Data Science** repository! 

This project is a daily commitment to bridging the gap between academic theory and real-world business application. Over the course of 100 days, I am translating the advanced concepts learned during my M.Tech degree into practical, code-driven solutions.

## 🎯 The Goal
Instead of following generic tutorials, every single day features a **unique business problem**. The focus is on writing clean, well-documented code to wrangle messy data, train predictive models, and extract actionable insights. 

## 📚 The 100-Day Curriculum 

### Phase 1: Data Foundation & Wrangling (Days 1 - 20)
*Focus: Python, Pandas, SQL, Missing Data, EDA, and Feature Engineering.*
* **Days 1-5:** Data Ingestion, Corruption Handling, and Imputation.
* **Days 6-10:** Relational Data (Joins/Merges), Aggregation, and Outlier Detection.
* **Days 11-15:** String Manipulation, Regex, and Datetime Engineering.
* **Days 16-20:** Exploratory Data Analysis (EDA) and Statistical Visualization.

### Phase 2: Machine Learning Core (Days 21 - 45)
*Focus: Scikit-learn, Regression, Classification, Clustering, and Evaluation.*
* **Days 21-25:** Linear & Logistic Regression (Predicting Prices and Binary Outcomes).
* **Days 26-30:** Decision Trees & Random Forests (Feature Importance & Non-linear data).
* **Days 31-35:** Support Vector Machines (SVM) & K-Nearest Neighbors (KNN).
* **Days 36-40:** Unsupervised Learning (K-Means Clustering & PCA for dimensionality reduction).
* **Days 41-45:** Model Tuning (GridSearch, Cross-Validation, and handling class imbalance).

### Phase 3: Deep Learning Architectures (Days 46 - 65)
*Focus: TensorFlow/Keras, PyTorch, ANNs, and Computer Vision.*
* **Days 46-50:** Artificial Neural Networks (ANNs) for tabular data.
* **Days 51-55:** Convolutional Neural Networks (CNNs) for image classification.
* **Days 56-60:** Transfer Learning (VGG16, ResNet) for complex visual tasks.
* **Days 61-65:** Model checkpointing, dropout, and optimizing neural architectures.

### Phase 4: Natural Language Processing - NLP (Days 66 - 80)
*Focus: NLTK, spaCy, Text Preprocessing, RNNs, and LSTMs.*
* **Days 66-70:** Tokenization, Stemming, Lemmatization, and TF-IDF.
* **Days 71-75:** Word Embeddings (Word2Vec) and Sentiment Analysis.
* **Days 76-80:** Sequence Modeling (RNNs, LSTMs, and BiLSTMs) for text generation and classification.

### Phase 5: Big Data & Advanced Systems (Days 81 - 100)
*Focus: PySpark, Streaming concepts, Recommender Systems, and AI.*
* **Days 81-85:** Big Data manipulation using Apache Spark (PySpark).
* **Days 86-90:** Recommender Systems (Collaborative & Content-based filtering).
* **Days 91-95:** Introduction to streaming data concepts (Simulating Kafka streams).
* **Days 96-100:** Capstone mini-projects combining ML pipelines with final business reports.

## 📂 Repository Structure
Each day has its own dedicated folder containing:
1. `problem_statement.md`: The business scenario and objective.
2. `dataset.csv`: The raw (often messy) data.
3. `solution.ipynb` (or `.py`): My step-by-step coded solution with technical commentary.
